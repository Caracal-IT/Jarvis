# Style Guide

This directory contains UX and design system documentation for the Jarvis application.

## Files

- **`ux-style-guide.md`** — Complete visual design system including colors, typography, spacing, and component specifications

## Purpose

The style guide ensures consistent visual presentation across all features of the Jarvis application. It defines the Iron Man-inspired design language and provides specific implementation guidance for developers.

## Usage

- **Specs define WHAT** — Feature requirements and user goals live in `docs/specs/`
- **Style guide defines HOW** — Visual appearance and UX details live here

When implementing UI features:
1. Read `docs/specs/` for functional requirements
2. Read `docs/style-guide/ux-style-guide.md` for visual implementation details
3. Ensure all colors, dimensions, and strings use resource files

## Relationship to Other Documentation

- **Standards** (`docs/standards/`) — Coding practices, architecture, licenses
- **Specs** (`docs/specs/`) — High-level feature requirements and acceptance criteria  
- **Style Guide** (this directory) — UX-specific colors, typography, spacing, components
